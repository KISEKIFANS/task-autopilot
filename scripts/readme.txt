#每一行的格式：任务序号,待匹配图片,匹配成功时的动作,匹配失败时的动作,时间,任务名
#	任务序号：每一行一个序号，建议间隔大一些，方便后续中间插入任务
#	待匹配图片：有两种输入
#		图片：即要匹配的图片
#		MATCH_YES：直接触发成功动作，此时失败动作无意义，可随意填
#		MATCH_NO：直接触发失败动作，此时成功动作无意义，可随意填
#	匹配成功时的动作：目前有三种，任务序号、鼠标动作、键盘动作
#		任务序号：跳转至指定任务
#		鼠标动作：可选click、*click、slip
#			click：左键单点，默认点击匹配图片中心，需要点击其他地方时使用click(x@y)。(x@y)为中心偏离坐标，(*x@*y)为绝对坐标
#           *click：按住alt再click，适用于需呼出鼠标的场景
#			slip_around：在图片中心周边矩形区域进行滑动，(x@y)为中心偏离距离，x为中心到矩形左右边距离，y为中心到矩形上下边距离
#		键盘动作：按键，若以*开头，表示调用pydirectinput而不是pyautogui，对游戏兼容更好，建议窗口焦点在游戏中时使用
#			支持组合键，如alt+f4，若调用pydirectinput应是*alt+*f4
#	匹配失败时的动作：目前有两种，任务序号、block
#		任务序号：跳转至指定任务
#		block：阻塞程序，等待用户排查问题
#	时间：当为图片时，表示最大允许的匹配失败次数；当为MATCH_YES或MATCH_NO时，表示延迟执行时间
#	任务名：仅起描述作用，对脚本无影响
